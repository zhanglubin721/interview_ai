## G1 垃圾回收器快，快在哪

内存结构的变化：由之前 Parallel GC 的物理分代变成了由多个同样大小区块内存组成的逻辑分代

部分 GC ：由于Parallel GC 是物理分代，不能实现扫描回收部分新生代和老年代，G1 可以（变相缩短了单次 STW 耗时），多轮回收，且通过负反馈调节机制根据例如上次扫描的区块数量、上次 GC 耗时等信息动态调整每次扫描的区块数量并且根据“收益/成本”性价比排序，优先回收性价比更高的区块，这也是 G1 名字的由来。在选定了要回收哪些区块之后根据 GC Root + Rset 避免了对老年代的全面扫描

可达性分析：GC Root 的定义没变，但是 G1 是完全并发，采用灰对象栈 + 工作窃取实现高效并行的可达性分析

内存分配：使用 bump—the—pointer + TLAB 对每个线程预分配区块空间，每个线程的内存存储在自己 TLAB 中，实现了高效的内存分配
